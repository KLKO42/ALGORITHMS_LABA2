Обоснование выбора структуры данных

Используется динамический массив, а не LinkedList по следующим причинам:

1. Операции, выполняемые с плейлистами, музыкой:
   - Добавление песен (функция insert_song).
   - Отображение всех треков с нумерацией (функция display_all)
   - Перемешивание (shuffle) — требуется доступ по индексу для обмена элементов!
   - Чтение и запись файла, которое требует последовательности.

2. Динамический массив куда более удобен, чем линковкой:
   - Доступ по индексу за O(1) — например, удаление по номеру (delete_song)
   - Перемешивание (random.shuffle)удобнее с массивом
   - Последовательный обход (display_all, filter_by_genre) хорошая локальность данных:)
   - Вставка в конец выполняется за O(1), т.е амортизированно

3. Если бы испрльзовался способ linkedlist, то столкнулась бы со следующими проблемами:
   - Для доступа по индексу (например, надо удалить трек под номером 16) потребовалось бы проходить список от начала за O(n)
   - Сложность в реализации перемешивания песен(если это вообще возможно)
   - Каждый элемент занимает дополнительную память под ссылки
